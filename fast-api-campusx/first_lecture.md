## üîë What is an API?
- **Definition**: A connector enabling communication between two software components (front-end ‚Üî back-end).  
- **Protocols**: Typically HTTP.  
- **Data Format**: JSON (universal, language-independent).  
- **Analogy**:  
  - Customer = Front-end  
  - Kitchen = Back-end  
  - Waiter = API  
  - Menu = Rules/protocols  
  - Plated food = Data format

## üèóÔ∏è Problems Solved by APIs

### 1. Pre-API Era (Monolithic Architecture)
- Front-end & back-end tightly coupled ‚Üí hard to share data.  
- Example: IRCTC train booking site couldn‚Äôt share data with apps like MakeMyTrip.  
- **Solution**:  
  - Decouple back-end.  
  - Add API layer with endpoints (e.g., `/trains`).  
  - External apps can access securely.  
  - Revenue opportunities via integrations.  

### 2. Smartphone Revolution
- Need for multiple front-ends (web, Android, iOS).  
- Monolithic approach ‚Üí duplicate codebases, high cost, sync issues.  
- **Solution**:  
  - Single database + single back-end.  
  - Expose via API.  
  - Multiple front-ends (web, Android, iOS) connect through API.  
  - Simplified, scalable architecture (used by Google, Uber, Zomato).
